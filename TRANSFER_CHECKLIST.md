## Pre-Build Requirements (Mac mini)

### 1. Xcode 26
Verify Xcode 26 is installed before running prebuild:
```bash
xcodebuild -version
```
Expected output should include "26" or higher.

### 2. Apple Developer Signing Certificates
Before building, set up your signing certificates in Xcode:
- Open Xcode → Settings → Accounts → Add your Apple Developer ID
- Configure your Team and signing certificates for local development
- Ensure "Sign to Run Locally" is enabled for your target bundle

### 3. Node.js
Verify Node.js 20+:
```bash
node -v
npm -v
```

## Build Commands (Run on Mac mini after file transfer)

### 1. Install dependencies
```bash
cd SiteDeckPro/mobile
npm install
```

### 2. Generate native iOS project (clean prebuild)
```bash
npx expo prebuild --clean
```
This will create the `ios/` directory from scratch using Expo SDK 55 templates with Xcode 26 support.

### 3. Install CocoaPods
```bash
cd ios
pod install
cd ..
```

### 4. Build for iOS Simulator (development)
```bash
npx expo run:ios --configuration Debug
```
Or for Release:
```bash
npx expo run:ios --configuration Release
```

### 5. Android (optional — eas.json still present for EAS Android builds)
```bash
npx expo run:android
```

## Notes

- **New Architecture is ON** in SDK 55 — `newArchEnabled` config option has been removed from Expo. It cannot be disabled. This is the final state.
- **expo-av removed** — no audio/video imports existed in the codebase. If audio/video features are added later, use `expo-audio` and `expo-video` (not expo-av).
- **No reanimated API migration needed** — the codebase uses no direct Reanimated APIs (`useSharedValue`, `useAnimatedStyle`, etc.). Only the babel plugin is used.
- **eas.json preserved** — Android EAS builds still work via the existing eas.json config. iOS builds will be done locally via `expo run:ios`, bypassing EAS.
- **android/ directory** — will be regenerated by `expo prebuild`. The existing directory can be kept or deleted.

## Verification Steps After Build

1. Launch the app in iOS Simulator
2. Verify Firebase Auth works (login flow)
3. Navigate to Home screen — check lesson stats load
4. Navigate to Tools menu — verify plan-gated tools render
5. Check that no red screen or crash occurs on initial load

If crashes occur, the most likely causes on a fresh prebuild:
- Firebase `google-services.json` / `GoogleService-Info.plist` not configured in Xcode
- Missing Apple signing entitlements
- Node modules mismatch — run `rm -rf node_modules && npm install` if needed
